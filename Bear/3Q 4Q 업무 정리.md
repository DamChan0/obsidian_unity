## LIDAR Parser

- 기존의 제조사에서 만든 data parser와 network connection sdk를 사용하는 것에 문제를 제기함
    
    - 코드의 가독성과 우리의 상황에 맞게 데이터를 처리하거나 전처리 또는 ethernet통신및 usb연결에 대한 변경이 용이 하지 않은 등 코드의 품질이 떨어지는 문제 있었음
        
    - servi q에 장착되는 pacecat LiDAR의 Driver를 직접 제작하였음(network 연결 , 데이터 parsing)
        
        - 데이터 파싱
            
            - 라이다 network packet을 parsing 하여 x,y point 좌표 데이터를 도출
                
            - 도출한 데이터를 ros /senser data format으로 변환하여 출력하여 rviz에서 까지 데이터 시각화
                
        - Network
            
            - destination IP와 port를 실행시 사용자가 설정 수 있도록하 높임
                
            - 제조사에서 제공한 sdk의 LiDAR IP주소를 변경하는 기능이 동작하지 않았음 이를 해결하기위해서 LiDAR의 IP 와 PORT를 세팅하는 명령 packet 구조를 분석하여 Parser 프로그램에 기능을 추가하였음

## servi q - AD display

- AD display 컴포넌트를 개발이 현재 touchscreen이 영상 렌더시 자원을 많이쓰고 있어서 최적화가 필요한점 하드웨어 적으로 세세한 제어가 필요한 점 그리고 새로운 장비를 추가한다는 점에서 RS의 일로 판단하여 기능 개발을 RS팀에서 할것을 제안하여 프로젝트를 주도함
    
- 빠르게 영상 출력및 초기 요구사항을 만족하여 성능 파악을 하였음
    
- 이를 통해 새로운 arm시스템을 사용하면 발생 하는 cpu 사용량 문제와 하드웨어 디코더 적용 방법 그리고 어떤 라이브러리들을 사용하는 것이 가장 좋은지 파악하여 다양한 테스트를 진행하여 빠르게 사용하는 기술 스택과 sw구조를 설계하였음
    
- 프로젝트를 원할하게 진행하기 위해서 참여하는 다른팀원들에게 업무를 할당하고 현상을 주기적으로 파악하기 위해서 데일리 스크럼을 하여 문제 상황이 길게 지속되는 것을 근본적으로 막음
    
- 새로운 시스템을 사용하면서 발생할 수 있는 빌드 시스템 문제를 선제적이고 근본적으로 인지하여 먼저 다른팀과 하이레벨 개발자들과 적극적으로 소통하면서 합의괸 해결 방법을 도출해내었고 이를 팀에 공유하여 개발이 다끝났는데 적용과 빌드등 운영과 관련해서 혼란이 일어날 것을 방지하였음
    
- SW설계 문서도 없고 요구사항의 정리된 문서도 없던 상황을 이슈화해서 PM에게 요구사항을 목록화 한 컨플루언스 문서를 만들어서 모두가 업데이트되는 것을 알수 있도록 해달라고 요청하여 실제로 적용이되어 모두가 볼 수 있게 하였고 sw구조 설계서 문서 양식을 자체 제작하여 [Design Doc](https://bearrobotics.atlassian.net/wiki/spaces/RS/pages/3708059753) [Template](https://bearrobotics.atlassian.net/wiki/spaces/RS/pages/3954245655) 프로젝트 인워들에게 공유하고 작성해줄 것을 요청하였르면 feature owner도 긍적적으로 수용하여 design 문서를 제가 만든 포맷과 초안을 사용하기로 하였음[[Servi Q] Ad_Display - software Architecture Design Document](https://bearrobotics.atlassian.net/wiki/spaces/RS/pages/3740303361)
    
    - 이를 통해서 더 체계적으로 회의와 논의를 진행하고 개발문서로 확보할 수 있도록 하였음

## 전반적인 태도

- 내 개발 스코프만 파는 것이 아니라 유니버스나 클라우드 인프라 로보틱스 등 다른 레이어의 작업과 이슈들도 이해하고 해결방법 까지 전달받은 다음 이해하기 위해서 노력하였음
    
- 이런 태로로 혼자서도 현재 이슈가 어디가 원인이고 어디서 해결이 가능할지 완벽하지는 않지만 가늠하고 PMTL에게 설명할 수 있는 능력을 갖추어 이슈을 등록과 할당 초기 처리까지 감당 할수 있는 수준의 역량을 키웠음
    
- 주어진 일을 하면서 비효율적인 구성이나 상황을 발견 할 시 팀내나 혼자서만 임시로 처리하는 것이 안니라 기술부채를 막고 회사가 더 나아가기 위해서 담당자들 또는 관련자들에게 공유및 문의하고 적극적으로 근본적인 해결책을 고안하고 이를 단순 논의가 아니라 실제 적용까지 이루어지도록 지속적으로 드라이브하고 신경씀
    
- 내가 알거나 도움을 줄수 이슈 라면 외면 하지 않고 적극적으로 찾아가서 도움을 주고 이것이 도움만 주어 계속 나만 할 수 있게 하는것이 아니라 문서를 작성하고 공유하여 모두가 같이 성장 을 도모 할 수 있도록 함
