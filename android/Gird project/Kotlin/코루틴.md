 - 비동기 작업을 간당하게 처리하기 위한 경량 스레드이다 
 - UI 스레드를 막지않고 시간이 걸리는 작업을 실행하기 위해서 사용된다. 

## 🧩 Section 1. 핵심 개념

| 요소                 | 역할           | 설명                                                                              |
| ------------------ | ------------ | ------------------------------------------------------------------------------- |
| **Coroutine**      | 비동기 코드 블록    | 실제 실행 단위. 일반 스레드보다 훨씬 가볍고 빠르게 생성·소멸됨                                            |
| **CoroutineScope** | 코루틴의 생명주기 관리 | 어떤 범위에서 코루틴이 실행될지 정의함 (`lifecycleScope`, `viewModelScope` 등)                    |
| **Dispatcher**     | 실행 스레드 지정    | `Dispatchers.Main` (UI), `Dispatchers.IO` (입출력), `Dispatchers.Default` (CPU 연산) |
| **suspend 함수**     | 일시 중단 가능한 함수 | 코루틴 내에서만 호출 가능. 다른 스레드로 넘어갔다가 결과를 기다릴 수 있음                                      |
| **launch / async** | 코루틴 실행 방식    | `launch`는 결과 없이 실행, `async`는 `await()`로 결과를 반환함                                 |

## ⚙️ Section 2. 주요 함수와 사용 시나리오

#### 1️⃣ `launch`

- 반환값이 필요 없는 작업에 사용 (예: 데이터 저장, UI 갱신)

```kotlin
lifecycleScope.launch {
    val images = database.imageDao().getAllImages() // suspend 함수
    photoList.clear()
    photoList.addAll(images)
    photoAdapter.notifyDataSetChanged()
}
```
